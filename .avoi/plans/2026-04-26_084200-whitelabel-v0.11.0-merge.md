## PHASE 2: FETCH UPSTREAM v0.11.0

### 2.1 Add upstream remote (if not already)
```bash
git remote add upstream https://github.com/avoi-ai/avoi-agent.git
```

### 2.2 Fetch full history (not shallow)
```bash
# If currently shallow, unshallow first
git fetch --unshallow upstream  # might be large
# OR fetch specific branch
git fetch upstream main
```

### 2.3 Identify merge base
```bash
git merge-base avoi/main upstream/main
```
This tells us where we diverged.

### 2.4 Try initial merge (dry run)
```bash
git checkout avoi/main
git merge upstream/main --no-commit --no-ff
```
See what conflicts come up. Then abort:
```bash
git merge --abort
```

## PHASE 3: MERGE WITH CONFLICT RESOLUTION

### 3.1 Conflict categories expected
1. **Branding conflicts** — where we changed "avoi" to "avoi" and upstream also changed the same lines (e.g. version strings, config keys)
2. **Structural conflicts** — files renamed upstream, or new files where we have parallel changes
3. **Clean merges** — most new v0.11.0 code should merge cleanly since we didn't touch it

### 3.2 Resolution strategy
For each conflicted file:
- **Functional changes**: Accept upstream (they have 12k more commits of bugfixes)
- **Branding changes**: Re-apply our AVOI branding after accepting upstream
- **Our exclusive code** (/avoi/ dir, MIGO TUI): Should not conflict since upstream doesn't have it

### 3.3 Automated conflict resolution
Create a script that:
1. Lists all conflicted files
2. For each file, checks if the conflict is only in branded strings
3. Uses `git checkout --ours` for /avoi/ dir files
4. For core files: merge manually with careful review

### 3.4 High-risk files requiring manual attention
- `run_agent.py` — core loop, upstream has transport layer changes
- `cli.py` — CLI has major changes
- `avoi_constants.py` — path resolution changes
- `agent/prompt_builder.py` — system prompt assembly
- `agent/memory_manager.py` — memory provider lifecycle
- `tools/registry.py` — tool registration changes
- `tools/terminal_tool.py` — terminal backend changes
- `tools/delegate_tool.py` — orchestrator role added
- `toolsets.py` — new tool definitions
- `model_tools.py` — tool discovery
